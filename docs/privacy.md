# Privacy And Data Boundary

Codex Swarm Monitor is a local-first development tool. It observes a selected local Codex workspace and renders local realtime activity in a browser served from `localhost`.

## What Stays Local

- Codex hook events
- tool names and command summaries
- Markdown file references
- workspace path and repository metadata
- Ralph/harness analysis
- runtime SQLite event history
- support bundles generated by the user

The app stores event history in a local SQLite database. It does not send workspace activity to a hosted service.

## Network Use

The normal runtime uses local loopback networking:

- `POST /events` receives Codex hook events from the local workspace.
- `GET /state` returns the selected workspace state.
- `GET /stream` uses workspace-scoped Server-Sent Events.
- `GET /avatar` generates local SVG portraits.

Standalone/plugin bootstrap scripts may download release archives and `.sha256` files from the configured GitHub release URL when a local runtime is not already installed. The checksum is verified before installation.

## Redaction

Common secret fields and token-looking values are redacted before persistence and before SSE broadcast, including API keys, bearer tokens, GitHub tokens, Slack tokens, passwords, cookies, and private keys.

## What Is Not Collected

The app does not intentionally collect:

- OpenAI API keys
- model prompts through a network proxy
- full file contents by default
- telemetry
- analytics
- remote avatar requests
- demo or seed events

Optional future LiteLLM/OpenInference model-call capture is explicitly outside the current default product boundary.

## User Control

Users can:

- choose the workspace folder
- install or uninstall project-local Codex hooks
- clear workspace-scoped events
- set local event retention
- export a local support bundle

Uninstalling hooks removes only Codex Swarm Monitor hook entries and `.codex/codex-swarm-monitor` runtime files from the selected workspace.
